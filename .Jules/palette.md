## 2024-06-22 - Pagination Navigation Accessibility
**Learning:** The default pagination controls in the application lacked structural semantics (using a generic `div` instead of `nav`), and icon-only buttons (`&laquo;` and `&raquo;`) missed `aria-label`s. Also, buttons denoting the current page lacked `aria-current="page"`, and ellipses buttons read awkwardly to screen readers without a null or specific `aria-label`.
**Action:** Ensure all pagination elements use a wrapping `<nav aria-label="Nawigacja po stronach">`. Provide text alternatives for purely visual symbol buttons (like `&laquo;` -> `Poprzednia strona`), dynamically set `aria-current="page"`, and explicitly set `type="button"` to avoid implicit form submissions, adhering to standard Polish accessibility patterns.
